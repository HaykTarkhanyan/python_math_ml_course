\section*{Problem 01 \texorpdfstring{$\,\cdot\,$}{-} Election Night Nail-Biter}
% ============================================================

\subsection*{Exercise}

A poll of $n = 900$ likely voters finds that 52\% support candidate A.

\begin{enumerate}[label=\alph*)]
  \item Build a 95\% \textbf{Wald} CI for the true proportion $p$. Based on this interval, can you call the election for candidate A?
  \item Now compute the \textbf{Wilson} CI for the same data. How does it differ from the Wald CI?
  \item Imagine only $n = 20$ voters were polled and 11 (55\%) said A. Recompute both Wald and Wilson CIs. Which one behaves better near the boundary, and why?
\end{enumerate}

\subsection*{Solution}

\paragraph{(a) Wald CI.}

The Wald 95\% CI for a proportion is $\hat{p} \pm z_{0.025}\,\text{SE}$, where
\[
  \text{SE}_{\text{Wald}} = \sqrt{\frac{\hat{p}(1-\hat{p})}{n}}
  = \sqrt{\frac{0.52 \cdot 0.48}{900}}
  = 0.01665.
\]
With $z_{0.025} = 1.96$:
\[
  \text{CI} = 0.52 \pm 1.96 \cdot 0.01665 = (0.4874,\; 0.5526).
\]

The lower bound $0.4874 < 0.5$, so the CI \textbf{contains} 0.5. We \textbf{cannot} call the election for candidate A at the 95\% confidence level -- the data is consistent with a tie or even a loss.

\paragraph{(b) Wilson CI.}

The Wilson CI is:
\[
  \frac{\hat{p} + \frac{z^2}{2n}}{1 + \frac{z^2}{n}}
  \;\pm\;
  \frac{z}{1 + \frac{z^2}{n}}
  \sqrt{\frac{\hat{p}(1-\hat{p})}{n} + \frac{z^2}{4n^2}}.
\]
For $n = 900$, $z^2/n = 3.8416/900 \approx 0.00427$, which is tiny. The Wilson CI evaluates to
\[
  (0.4873,\; 0.5525).
\]

At $n = 900$ the Wald and Wilson CIs are nearly identical -- the Wilson correction is negligible for large $n$. Both contain 0.5.

\paragraph{(c) Small sample: $n = 20$, $\hat{p} = 11/20 = 0.55$.}

\textbf{Wald:}
\[
  \text{SE} = \sqrt{\frac{0.55 \cdot 0.45}{20}} = 0.1112,
  \qquad
  \text{CI} = (0.3320,\; 0.7680).
\]

\textbf{Wilson:}
\[
  \text{CI} = (0.3421,\; 0.7418).
\]

The Wilson CI is narrower and pulled toward 0.5. The Wald CI can overshoot $[0, 1]$ for extreme $\hat{p}$ or small $n$ because it treats the sampling distribution as symmetric. The Wilson CI accounts for the curvature of the binomial likelihood and always stays within valid bounds, making it more reliable for small samples.

\section*{Problem 02 \texorpdfstring{$\,\cdot\,$}{-} A/B Test: Ship It or Wait?}
% ============================================================

\subsection*{Exercise}

An e-commerce company runs an A/B test on their checkout page.

\begin{itemize}
  \item Control ($n_1 = 200$): conversion rate $\hat{p}_1 = 0.08$ (8\%).
  \item Treatment ($n_2 = 200$): conversion rate $\hat{p}_2 = 0.11$ (11\%).
\end{itemize}

The product manager says ``3\% lift -- ship it!''

\begin{enumerate}[label=\alph*)]
  \item Build a 95\% CI for the difference $p_2 - p_1$.

  \emph{Recall:} $\text{SE} = \sqrt{\frac{\hat{p}_1(1-\hat{p}_1)}{n_1} + \frac{\hat{p}_2(1-\hat{p}_2)}{n_2}}$.
  \item Does the CI contain 0? What does this tell the PM?
  \item How large would each group need to be (equal sizes) so that the expected CI width is narrow enough to exclude 0, assuming the true lift really is 3\%?
\end{enumerate}

\subsection*{Solution}

\paragraph{(a) 95\% CI for $p_2 - p_1$.}

\[
  \hat{p}_2 - \hat{p}_1 = 0.03.
\]
\[
  \text{SE} = \sqrt{\frac{0.08 \cdot 0.92}{200} + \frac{0.11 \cdot 0.89}{200}}
  = \sqrt{\frac{0.0736}{200} + \frac{0.0979}{200}}
  = \sqrt{0.000857}
  = 0.02928.
\]
\[
  \text{CI} = 0.03 \pm 1.96 \cdot 0.02928 = (-0.0274,\; 0.0874).
\]

\paragraph{(b) Does the CI contain 0?}

\textbf{Yes.} Since the CI includes 0, we cannot rule out that the true conversion rates are equal. The observed 3\% lift is not statistically significant at the 95\% level. The PM should \textbf{not} ship based on this evidence alone -- either collect more data or accept the uncertainty.

\paragraph{(c) Required sample size.}

We need the margin of error to be less than the observed effect: $\text{ME} < 0.03$.
\[
  z \cdot \sqrt{\frac{p_1(1-p_1) + p_2(1-p_2)}{n}} < 0.03
\]
\[
  n \geq \frac{z^2 \bigl[p_1(1-p_1) + p_2(1-p_2)\bigr]}{0.03^2}
  = \frac{3.8416 \cdot (0.0736 + 0.0979)}{0.0009}
  = \frac{3.8416 \cdot 0.1715}{0.0009}
  \approx 733.
\]

Each group needs at least $n = 733$ users -- roughly $3.7\times$ the original 200.

\section*{Problem 03 \texorpdfstring{$\,\cdot\,$}{-} Bootstrap vs Formula}
% ============================================================

\subsection*{Exercise}

Generate $n = 30$ observations from $\text{Exp}(\lambda = 1)$ in Python (set \texttt{np.random.seed(509)}).

\begin{enumerate}[label=\alph*)]
  \item Compute the \textbf{analytical} SE of $\bar{X}$ (since $\sigma = 1/\lambda = 1$, this is $\sigma / \sqrt{n}$).
  \item Compute the \textbf{bootstrap} SE with $B = 10{,}000$ resamples.
  \item Build three 95\% CIs for $\mu = 1/\lambda$:
  \begin{enumerate}[label=(\roman*)]
    \item Normal-theory: $\bar{X} \pm z_{0.025} \cdot \text{SE}$
    \item Bootstrap percentile: $[\hat{\theta}^*_{0.025},\; \hat{\theta}^*_{0.975}]$
    \item $t$-interval: $\bar{X} \pm t_{n-1, 0.025} \cdot \frac{s}{\sqrt{n}}$
  \end{enumerate}
  Compare widths. Do all three contain the true $\mu = 1$?
  \item Repeat (a)-(c) for the \textbf{sample median} instead of the mean. Which method \textbf{cannot} give you a formula-based SE?
\end{enumerate}

\subsection*{Solution}

See the accompanying Jupyter notebook (\texttt{hw\_07\_08\_solutions.ipynb}) for runnable code and plots.

With seed 509: $\bar{X} = 0.8061$, $s = 0.8601$, median $= 0.5374$.

\paragraph{(a) Analytical SE.}

For $\text{Exp}(\lambda = 1)$, $\sigma = 1/\lambda = 1$, so
\[
  \text{SE}(\bar{X}) = \frac{\sigma}{\sqrt{n}} = \frac{1}{\sqrt{30}} = 0.1826.
\]

\paragraph{(b) Bootstrap SE.}

$\text{SE}_{\text{boot}} = 0.1531$, close to the analytical value. The bootstrap SE uses the sample's actual distribution rather than the known $\sigma$, so it reflects the randomness in this particular sample.

\paragraph{(c) Three 95\% CIs for $\mu = 1$.}

\begin{center}
\begin{tabular}{lcc}
\hline
\textbf{Method} & \textbf{95\% CI} & \textbf{Width} \\
\hline
Normal-theory ($z$) & $(0.4483,\; 1.1640)$ & $0.7157$ \\
Bootstrap percentile & $(0.5288,\; 1.1249)$ & $0.5961$ \\
$t$-interval & $(0.4850,\; 1.1273)$ & $0.6424$ \\
\hline
\end{tabular}
\end{center}

All three contain $\mu = 1$. The normal-theory interval is widest because it uses the theoretical $\sigma = 1$, which overestimates the SE for this particular sample (where $s = 0.86$). The $t$-interval uses $s$ but inflates via $t_{29} = 2.045 > 1.96$. The bootstrap percentile is narrowest as it adapts to the sample's actual shape.

\paragraph{(d) Sample median.}

There is \textbf{no closed-form formula} for the SE of the median of an exponential distribution (in general, the median's SE depends on the density at the population median, which requires knowing the distribution). The bootstrap is the only practical option here.

Bootstrap SE of the median: $0.1968$. Bootstrap percentile CI: $(0.2412,\; 0.9499)$.

The true median of $\text{Exp}(1)$ is $\ln 2 \approx 0.6931$, which falls inside this interval.

The normal-theory and $t$-interval methods \textbf{cannot} be directly applied to the median because they require a formula for the standard error. This is precisely the scenario where the bootstrap shines.

\section*{Problem 04 \texorpdfstring{$\,\cdot\,$}{-} Bootstrap the Correlation}
% ============================================================

\subsection*{Exercise}

Given these $(x, y)$ pairs:
\[
  (1,2),\; (2,3),\; (3,5),\; (4,4),\; (5,7),\; (6,8),\; (7,6),\; (8,9),\; (9,10),\; (10,12).
\]

\begin{enumerate}[label=\alph*)]
  \item Compute the Pearson correlation $r$.
  \item Use the bootstrap ($B = 5{,}000$) to build a 95\% percentile CI for the population correlation $\rho$.
  \item Plot the bootstrap distribution of $r^*$. Is it symmetric? If not, why might that be?
  \item Fisher's $z$-transform gives an analytical CI: transform $z = \tfrac{1}{2}\ln\tfrac{1+r}{1-r}$, build a normal CI using $\text{SE}(z) = \tfrac{1}{\sqrt{n-3}}$, then back-transform with $r = \tfrac{e^{2z}-1}{e^{2z}+1}$. Compare with your bootstrap CI.
\end{enumerate}

\subsection*{Solution}

See the notebook for code and histogram of the bootstrap distribution.

\paragraph{(a) Pearson $r$.}

$r = 0.9506$ -- a strong positive linear relationship.

\paragraph{(b) Bootstrap 95\% percentile CI.}

Bootstrap 95\% CI: $(0.8250,\; 0.9928)$.

\paragraph{(c) Symmetry of bootstrap distribution.}

\textbf{No.} The distribution is strongly left-skewed (skewness $\approx -5.0$). This happens because $r$ is bounded by $[-1, 1]$. When the true correlation is high ($\rho \approx 0.95$), there is very little room to go higher but plenty of room to go lower, creating a long left tail. This ``ceiling effect'' is exactly why Fisher's $z$-transform exists.

\paragraph{(d) Fisher $z$-transform CI.}

\[
  z = \tfrac{1}{2}\ln\frac{1+r}{1-r}
  = \tfrac{1}{2}\ln\frac{1.9506}{0.0494}
  = 1.8384.
\]
\[
  \text{SE}(z) = \frac{1}{\sqrt{n-3}} = \frac{1}{\sqrt{7}} = 0.3780.
\]
\[
  z \in (1.8384 - 1.96 \cdot 0.3780,\; 1.8384 + 1.96 \cdot 0.3780) = (1.0976,\; 2.5792).
\]

Back-transform: $r = \frac{e^{2z} - 1}{e^{2z} + 1}$:
\[
  \rho \in (0.7996,\; 0.9886).
\]

The Fisher CI is wider on the left (lower bound 0.80 vs bootstrap's 0.83), reflecting the fact that the $z$-transform properly symmetrizes the distribution. Both methods agree that $\rho$ is strongly positive. The Fisher CI is slightly more conservative -- appropriate given the small sample size ($n = 10$).

\section*{Problem 05 \texorpdfstring{$\,\cdot\,$}{-} When Bootstrap Breaks}
% ============================================================

\subsection*{Exercise}

Consider $X_1, \ldots, X_n \overset{\text{i.i.d.}}{\sim} \text{Uniform}(0, \theta)$. The MLE is $\hat{\theta} = X_{(n)} = \max(X_i)$.

\begin{enumerate}[label=\alph*)]
  \item Show that $n(\theta - X_{(n)}) \xrightarrow{d} \text{Exp}(1/\theta)$.

  \emph{Hint: start from $P(X_{(n)} \leq x) = (x/\theta)^n$ and substitute $u = n(\theta - x)$.}
  \item Using $n = 50$ and $\theta = 1$, simulate 10,000 samples. For each sample, also run $B = 1{,}000$ bootstrap resamples of $\hat{\theta}^*$. Compare the true distribution of $n(\theta - \hat{\theta})$ with the bootstrap distribution of $n(\hat{\theta} - \hat{\theta}^*)$. Do they match?
  \item Explain why the bootstrap fails here. What is special about the convergence rate of $\hat{\theta}$?

  \emph{Hint: the bootstrap ``works'' when the convergence rate is $\sqrt{n}$. Here it is $n$.}
\end{enumerate}

\subsection*{Solution}

\paragraph{(a) Limiting distribution of $n(\theta - X_{(n)})$.}

The CDF of the maximum is:
\[
  P(X_{(n)} \leq x) = \left(\frac{x}{\theta}\right)^n, \qquad 0 \leq x \leq \theta.
\]

Let $U = n(\theta - X_{(n)})$, so $X_{(n)} = \theta - U/n$. Then for $u \geq 0$:
\[
  P(U \leq u) = P\!\bigl(X_{(n)} \geq \theta - u/n\bigr)
  = 1 - P\!\bigl(X_{(n)} < \theta - u/n\bigr)
  = 1 - \left(\frac{\theta - u/n}{\theta}\right)^n
  = 1 - \left(1 - \frac{u}{n\theta}\right)^n.
\]

Taking $n \to \infty$:
\[
  \left(1 - \frac{u}{n\theta}\right)^n \to e^{-u/\theta},
\]
so
\[
  P(U \leq u) \to 1 - e^{-u/\theta},
\]
which is the CDF of $\text{Exp}(\text{rate} = 1/\theta)$, i.e., $\text{Exp}$ with mean $\theta$. $\qed$

\paragraph{(b) Simulation.}

See the notebook for code and overlay histograms. The two distributions look very different:
\begin{itemize}
  \item The \textbf{true distribution} of $n(\theta - \hat{\theta})$ is approximately $\text{Exp}(1)$ -- a smooth exponential curve on $[0, \infty)$.
  \item The \textbf{bootstrap distribution} of $n(\hat{\theta} - \hat{\theta}^*)$ has a point mass at 0 (when the bootstrap resample includes the original maximum) and decays differently. It does \textbf{not} match the true distribution.
\end{itemize}

\paragraph{(c) Why does the bootstrap fail?}

The bootstrap works by approximating the true sampling distribution through resampling. Its validity relies on the \textbf{convergence rate being $\sqrt{n}$}, so that the plug-in distribution and the true distribution are close in a precise sense (the bootstrap is ``consistent'').

For $\hat{\theta} = X_{(n)}$, the convergence rate is $n$, not $\sqrt{n}$. This is a \textbf{super-efficient} estimator -- it converges faster than the usual parametric rate.

The key problem: the bootstrap resamples from $\{X_1, \ldots, X_n\}$, so $\hat{\theta}^* = \max(X_i^*) \leq \hat{\theta}$ \textbf{always}. This means $\hat{\theta}^* = \hat{\theta}$ with probability $1 - (1-1/n)^n \approx 1 - 1/e \approx 0.632$. The bootstrap distribution has a large point mass at $\hat{\theta}$, while the true distribution of $\hat{\theta}$ has no point mass at $\theta$. The bootstrap fundamentally cannot reproduce the behavior near the boundary, so it gives invalid confidence intervals.

\textbf{Remedy:} For boundary estimators like this, use parametric bootstrap (simulate from $\text{Uniform}(0, \hat{\theta})$) or exact methods based on the known limiting distribution $\text{Exp}(1/\theta)$.
