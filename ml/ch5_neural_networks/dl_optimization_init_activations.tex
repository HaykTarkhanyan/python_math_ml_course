\begin{frame}{Which activation? The short answer}
  LMU showed the full zoo -- sigmoid, tanh, ReLU, Leaky ReLU, PReLU, ELU, SELU. In practice:
  \vfill
  \begin{itemize}\small
    \item \textbf{Hidden layers: ReLU by default.} Its derivative is $1$ for $z>0$, so gradients
      don't vanish, and it is cheap. Reach for \textbf{Leaky ReLU / GELU / ELU} only if you hit
      \emph{dying ReLUs}.
    \item \textbf{Output layer: match the task} -- \textbf{identity} for regression,
      \textbf{sigmoid} for binary, \textbf{softmax} for multiclass.
    \item \textbf{Sigmoid/tanh in hidden layers: avoid} -- they saturate and vanish the gradient
      (the wk18 recursion, made concrete).
  \end{itemize}
  \vfill
  \fcolorbox{paramgreen}{paramgreen!8}{\parbox{0.97\linewidth}{\small
    Output activation $+$ loss are two sides of one coin (maximum likelihood): sigmoid$+$cross-entropy,
    softmax$+$cross-entropy, identity$+$MSE -- which is why the output gradient is the clean
    residual $f-y$.}}
\end{frame}

\begin{frame}{The whole chapter in one arc}
  \begin{itemize}\small
    \item \textbf{A neuron} $=$ affine $+$ activation $=$ a regression you know. \textbf{Hidden
      layers} learn a feature space so the boundary can bend (representation learning).
    \item \textbf{Depth \& width} make it expressive (universal approximation, folding);
      \textbf{softmax} handles many classes.
    \item \textbf{Backprop} $=$ the chain rule backward once, reusing shared error signals -- it
      gives every gradient in a single pass.
    \item \textbf{Regularization} (weight decay, early stopping, dropout, augmentation) fights
      overfitting; \textbf{optimization} (SGD$+$momentum / Adam, schedules, init, BatchNorm)
      makes training fast and stable.
  \end{itemize}
  \vfill
  \fcolorbox{paramgreen}{paramgreen!8}{\parbox{0.97\linewidth}{\small
    Every giant model from the intro deck -- LLMs, diffusion, AlphaFold -- is this same
    machinery, scaled up and specialized. You now know how it is built and trained.}}
\end{frame}

% ---------------------------------------------------------------
\begin{frame}{Where next: architectures shaped to the data}
  The MLP treats every input feature the same. Real signals have \textbf{structure} a plain MLP
  wastes:
  \vfill
  \begin{itemize}\small
    \item \textbf{Images} have \emph{locality} and \emph{translation} structure $\to$
      \textbf{convolutional networks (CNNs)}.
    \item \textbf{Sequences} (text, audio, time series) have \emph{order} and \emph{memory} $\to$
      \textbf{recurrent nets / transformers}.
  \end{itemize}
  \vfill
  \fcolorbox{armblue}{armblue!8}{\parbox{0.97\linewidth}{\small
    Same neurons, same backprop, same optimizers -- but the \emph{architecture} is shaped to the
    data. \textbf{Next chapter: convolutional neural networks.}}}
\end{frame}
