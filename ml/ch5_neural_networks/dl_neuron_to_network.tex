\begin{frame}{See it learn: play with it}
  The single best way to internalise this chunk is to \textbf{watch the boundary bend} live:
  \vfill
  \begin{itemize}\small
    \item \textbf{TensorFlow Playground} -- \texttt{playground.tensorflow.org}. Pick the
      \emph{XOR} or \emph{two-spirals} data. Start with \textbf{0 hidden layers} (it fails,
      like a single neuron), then add \textbf{one hidden layer} and watch it become separable.
      Toggle \textbf{ReLU vs sigmoid} and add neurons.
    \item \textbf{3Blue1Brown -- ``Neural networks''} (chapters 1--4). The clearest visual
      intuition for what the weights are doing.
    \item \textbf{mlu-explain.github.io/neural-networks} -- a short, clickable explainer.
    \item \textbf{Karpathy, ``Neural Networks: Zero to Hero''} (\texttt{micrograd}). Builds a
      neuron -- and the training that comes next -- from scratch in Python.
  \end{itemize}
  \vfill
  \fcolorbox{armblue}{armblue!8}{\parbox{0.97\linewidth}{\small
    Try to \textbf{predict} before you press play: how many hidden neurons does XOR need? What
    happens with \emph{zero} hidden layers? Twenty minutes here beats an hour of slides.}}
\end{frame}

% ---------------------------------------------------------------
\begin{frame}{Recap: from a neuron to a network}
  \begin{itemize}\small
    \item A \textbf{neuron} $=$ affine map $\mathbf{w}^\top\mathbf{x}+b$, then an activation
      $\tau$. Choose $\tau$ and you recover a model you know: identity $\to$ linear regression,
      sigmoid $\to$ logistic regression.
    \item \textbf{One neuron draws one straight boundary.} That is why it cannot solve XOR, or
      any curved problem.
    \item A \textbf{hidden layer learns a new feature space} -- it re-coordinatises the data
      (XOR's points collapsed so a straight cut works). This is \textbf{representation
      learning}.
    \item More hidden neurons / layers $=$ richer feature spaces $=$ more flexible boundaries.
  \end{itemize}
  \vfill
  \fcolorbox{paramgreen}{paramgreen!8}{\parbox{0.97\linewidth}{\small
    Everything here is still ``a stack of affine maps and nonlinearities'' -- exactly the idea
    behind the huge models in the intro deck, just tiny.}}
\end{frame}

% ---------------------------------------------------------------
\begin{frame}{The open question: who picks the weights?}
  \begin{columns}[T]
    \column{0.6\textwidth}
    In the XOR and hidden-layer examples, the weights were \textbf{guessed} for us. That will
    not scale:
    \begin{itemize}\small
      \item real networks have \textbf{millions} of weights -- nobody sets them by hand;
      \item once there is a hidden layer the loss is \textbf{no longer convex}, so there is no
        neat closed-form solution and no single guaranteed minimum.
    \end{itemize}
    \vskip 4pt
    So how do we \emph{learn} good weights automatically?
    \column{0.38\textwidth}
    \fcolorbox{armorange}{armorange!12}{\parbox{0.95\linewidth}{\small
      \textbf{The answer is training.} \textbf{Gradient descent} moves the weights downhill and
      \textbf{backpropagation} gets every gradient in one pass -- the algorithm that makes deep
      learning possible. We get there later in the chapter, after first generalizing the
      architecture (deeper nets, many classes).}}
  \end{columns}
\end{frame}
