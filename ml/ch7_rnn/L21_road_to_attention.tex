\begin{frame}{The cell state}
  \begin{columns}[T]
    \column{0.42\textwidth}
    \small
    The LSTM adds a second track running alongside the hidden state $\mathbf{z}$: the
    \textbf{cell state} $\mathbf{c}$. At every step it is only lightly modified - a
    little is removed, a little is added.
    \vskip 6pt
    \fcolorbox{popblue}{popblue!6}{\parbox{0.95\linewidth}{\small
      \textbf{Sorting-line metaphor} (ch1-6's world): the belt itself carries context
      past every station; each worker only adds or removes a little as the box goes
      by. Nobody rebuilds the box from scratch at every station.}}
    \vskip 6pt
    {\footnotesize Every horizontal line in the diagram is a full vector; circles are
    element-wise operations; boxes are learned layers.}
    \column{0.58\textwidth}
    \begin{center}
      \includegraphics[width=\linewidth]{olah_lstm_chain.png}
    \end{center}
  \end{columns}
\end{frame}

\begin{frame}{Anatomy of one LSTM step}
  \footnotesize
  Same toy setup as L20's anatomy frame: a 3-word vocabulary, now a \textbf{2-d cell
  state} alongside the 2-d hidden state. Three gates and one candidate, each a function
  of $\mathbf{x}^{[t]}$ and $\mathbf{z}^{[t-1]}$; each gate is a sigmoid - a vector in
  $(0,1)^2$ used as an \textbf{element-wise mask} ($\odot$).
  \vskip 3pt
  \begin{center}
    \resizebox{0.98\linewidth}{!}{%

        cell/.style={draw, rounded corners, align=center, minimum height=0.8cm,

      \node[armblue, anchor=east] at (-1.4,3.0) {$\mathbf{c}^{[t-1]}\in\R^2$};
      \node[armblue, anchor=west] at (10.7,3.0) {$\mathbf{c}^{[t]}\in\R^2$};
      \node[armblue, font=\tiny] at (4.6,3.4) {cell highway - modified lightly, additively};

      \node[draw, circle, fill=armred!18, minimum size=0.6cm] (fmul) at (1.2,3.0) {$\odot$};

\begin{frame}{For the record: the four equations}
  \footnotesize\textcolor{gray}{Reference, not exam material - shown once.}
  \vskip 2pt
  \begin{center}
    \begin{tikzpicture}[scale=0.86, transform shape]
      \node[draw=armred, fill=armred!8, rounded corners=6pt, text width=11cm,
        align=left, inner sep=9pt, font=\normalsize] at (0, 3) {
        \textbf{\textcolor{armred}{Forget gate:}} \quad
        $\mathbf{f}^{[t]} = \sigma\!\big(\mathbf{W}_f^\top\xv^{[t]} + \mathbf{V}_f^\top\mathbf{z}^{[t-1]} + \mathbf{b}_f\big)$
      };
      \node[draw=paramgreen, fill=paramgreen!8, rounded corners=6pt, text width=11cm,
        align=left, inner sep=9pt, font=\normalsize] at (0, 1.6) {
        \textbf{\textcolor{paramgreen}{Input gate:}} \quad
        $\mathbf{i}^{[t]} = \sigma\!\big(\mathbf{W}_i^\top\xv^{[t]} + \mathbf{V}_i^\top\mathbf{z}^{[t-1]} + \mathbf{b}_i\big)$
        \\[3pt] \phantom{Input gate:} \quad
        $\tilde{\mathbf{c}}^{[t]} = \tanh\!\big(\mathbf{W}^\top\xv^{[t]} + \mathbf{V}^\top\mathbf{z}^{[t-1]} + \mathbf{b}\big)$
      };
      \node[draw=armblue, fill=armblue!8, rounded corners=6pt, text width=11cm,
        align=left, inner sep=9pt, font=\normalsize] at (0, 0) {
        \textbf{\textcolor{armblue}{Cell update:}} \quad
        $\mathbf{c}^{[t]} = \mathbf{f}^{[t]} \odot \mathbf{c}^{[t-1]} \;+\; \mathbf{i}^{[t]} \odot \tilde{\mathbf{c}}^{[t]}$
      };
      \node[draw=armorange, fill=armorange!8, rounded corners=6pt, text width=11cm,
        align=left, inner sep=9pt, font=\normalsize] at (0, -1.5) {
        \textbf{\textcolor{armorange}{Output gate:}} \quad
        $\mathbf{o}^{[t]} = \sigma\!\big(\mathbf{W}_o^\top\xv^{[t]} + \mathbf{V}_o^\top\mathbf{z}^{[t-1]} + \mathbf{b}_o\big)$
        \quad $\mathbf{z}^{[t]} = \mathbf{o}^{[t]} \odot \tanh(\mathbf{c}^{[t]})$
      };
      \node[draw=popblue, fill=popblue!6, rounded corners=6pt, text width=12.5cm,
        align=center, inner sep=8pt, font=\small] at (0, -3.2) {
        The punchline: $\mathbf{c}^{[t]} = \mathbf{f}^{[t]}\odot\mathbf{c}^{[t-1]} + \dots$
        is \textbf{additive} - gradients flow through the cell state with minimal
        decay. ResNet solved \emph{depth} with addition in 2015; LSTM solved
        \emph{time} with addition in 1997 - same medicine, 18 years earlier.
      };
    \end{tikzpicture}
  \end{center}
\end{frame}

\begin{frame}{Does it work? Proof + price}
  \begin{columns}[T]
    \column{0.56\textwidth}
    \begin{center}
      \includegraphics[width=\linewidth]{gradient_flow_comparison.pdf}
    \end{center}
    \column{0.44\textwidth}
    \footnotesize
    Same toy recall task as L20's empirical proof, same measurement method
    (\texttt{RNNCell}/\texttt{LSTMCell} unroll, per-step \texttt{retain\_grad()}).
    \vskip 2pt
    \fcolorbox{armred}{armred!8}{\parbox{0.95\linewidth}{\footnotesize
      \textbf{Honestly reported}: at this budget the measured gap is not yet
      dramatic - untrained gates start close to "half open," and a hard recall task
      needs real training before the additive path pays off. The
      \emph{architectural} argument (addition, not repeated multiplication) does not
      depend on this toy net converging.}}
    \vskip 3pt
    \textbf{Price:} 4 weight sets instead of 1. Our $d_x{=}1,d_h{=}16$ vanilla cell:
    $16{+}256{+}16=288$ params. LSTM: $4\times288=\mathbf{1{,}152}$.
    \vskip 2pt
    \textbf{GRU} (Cho et al., 2014): a streamlined 3-gate cousin, similar results, a
    common default when 1,152 is too many.
  \end{columns}
\end{frame}

\begin{frame}{The tokenization trilemma}
  \small
  The review sentence, split three ways:
  \begin{center}

  \end{center}
  \fcolorbox{popblue}{popblue!6}{\parbox{0.97\linewidth}{\footnotesize

\begin{frame}{One-hot and its poverty}
  \begin{columns}[T]
    \column{0.42\textwidth}
    \small
    Each token gets an ID; each ID, a one-hot vector. Callback: [03]'s one-hot

    \vskip 6pt

      geometry carries zero meaning.}}

\begin{frame}{Embeddings}
  \begin{columns}[T]
    \column{0.42\textwidth}
    \small

    word's own vector. An \textbf{embedding table} is just that $\mathbf{W}$, grown to

    \fcolorbox{popblue}{popblue!6}{\parbox{0.95\linewidth}{\small

    \vskip 4pt
    {\footnotesize Trained once on huge corpora (word2vec, GloVe) and reused - a warm
    start, i.e. L18's transfer learning, for text. Limitation in one line: "bank" gets
    ONE vector regardless of context - the GenAI chapter scratches that itch. Explore:

\begin{frame}{The objective}
  \begin{columns}[T]
    \column{0.46\textwidth}
    \fcolorbox{popblue}{popblue!6}{\parbox{0.95\linewidth}{\footnotesize
      $$P(y_1,\dots,y_T) = \prod_{i=1}^{T} P(y_i \mid y_1,\dots,y_{i-1})$$}}
    \vskip 6pt
    \small
    Turns raw text into supervised data \textbf{for free}: every prefix is an input,
    every next token a label. No labeling budget, no annotators - just more text.
    \vskip 6pt
    \fcolorbox{paramgreen}{paramgreen!8}{\parbox{0.95\linewidth}{\small
      Said slowly, once: this exact objective, scaled up, is what GPT trains on.}}
    \column{0.54\textwidth}
    \begin{center}
      \begin{tikzpicture}[font=\scriptsize, >=Latex, node distance=6pt]
        \node[draw, rounded corners, fill=armblue!10, align=left] (s) at (0,3.2)
          {"The pomegranates arrived fresh"};

        \node[draw, rounded corners, fill=armblue!6, align=left] (p2) at (0,1.5) {The pomegranates $\to$ arrived};
        \node[draw, rounded corners, fill=armblue!6, align=left] (p3) at (0,0.8) {The pomegranates arrived $\to$ fresh};

\begin{frame}{Encoder-decoder}
  \begin{columns}[T]
    \column{0.42\textwidth}
    \small
    Translation needs different input/output lengths. Fix: an \textbf{encoder} LSTM
    reads the source and hands ONE \textbf{context vector} $C$ to a \textbf{decoder}
    LSTM that writes the target.
    \vskip 6pt
    \fcolorbox{popblue}{popblue!6}{\parbox{0.95\linewidth}{\small
      Running example: "I watch all of this in silence, and the connoisseurs speak
      inside me." goes in - and the decoder writes out the chapter's Armenian line,
      the sentence you just watched cost 104 tokens.}}
    \column{0.58\textwidth}
    \begin{center}
      \includegraphics[width=\linewidth]{alammar_seq2seq_still.png}
    \end{center}
  \end{columns}
\end{frame}

\section*{Recap}

\begin{frame}{Recap}
  \begin{itemize}
    \item Length varies, memory fades (L20) $\to$ LSTM fixes fading with an
      \textbf{additive} cell state - same medicine as ResNet, 18 years earlier - at
      4$\times$ the parameter cost.
    \item Text becomes numbers through \textbf{tokenization}: char / word / subword,
      and the Armenian line costs several times more tokens than its English gloss.
    \item One-hot is geometry-blind; \textbf{embeddings} are one-hot's learned,
      dense upgrade - the anatomy frame's "row of $\mathbf{W}$," grown up.
    \item One objective - \textbf{predict the next token} - powers a char-LSTM
      writing Armenian and, at a different scale, GPT.
    \item \textbf{Seq2seq} squeezes an entire source sentence into one vector - and
      even the LSTM's fix for fading memory cannot fix a memory that is
      \emph{compressed}.
  \end{itemize}
  \vskip 6pt
  \fcolorbox{paramgreen}{paramgreen!8}{\parbox{0.97\linewidth}{
      \textbf{Next:} the decoder wants to look back at everything. Letting it do
      that - attention - is where the GenAI chapter begins.}}
\end{frame}
