Composition vs Inheritance

In object‚Äêoriented design, composition over inheritance is the principle that you should prefer to build complex behavior by combining (‚Äúcomposing‚Äù) simple objects, rather than by creating deep class hierarchies through inheritance.

class Bird:
    def fly(self):  
        print("Flapping wings")

class Duck(Bird):
    pass

class Ostrich(Bird):
    # Ostriches can't fly, but still inherit fly()
    def fly(self):
        raise NotImplementedError("Ostrich can't fly!")

Here, Ostrich must override fly() just to prevent inherited behavior that doesn‚Äôt apply. As more behaviors accumulate (swimming, singing, nesting‚Ä¶), you could end up with a combinatorial explosion of subclasses.
